Search and Rescue UAV - Conceptual Design
Advisors: Prof. M. Ramakrishna and B. Govindarajan

Every year a large number of people get lost in forests and many of them lose their lives. Search and Rescue teams on foot are vastly insufficient for larger forests or in difficult to navigate terrains. Therefore, we designed a UAV that can perform this task.

First, it performs the search task and locates the rescuee using a state-of-the-art Computer Vision algorithm. It also carries water, energy dense foods and drinks, and a medical kit.

The design is an electric fixed wing configuration with 4 rotors providing VTOL capabilities.

Max. Takeoff Weight - 10.47 kg
Payload Capacity - 1.3 kg
Range - 115 km
Cruise speed - 20 m/s

Search and Rescue UAV - Structural Design
Advisors: Prof. H.S.N. Murthy and S. Murugan

Fabricating the UAV using the design from the conceptual design phase. Choosing the right materials, designing the structure against failure and then fabricating the final product.

Material Testing: Tested Al-6082 using a UTM. The properties were found out to be: Yield Stress: 71 MPa, Tensile Stress: 320 MPa, Young's Modulus: 84.66 GPa.

Structural Design: Calculating wing (skin, spar, stringers, ribs) and fuselage (bulkhead) component dimensions for design against failure. I have written a computer code that produces a fine discretization of the wing skin and spar and calculates stresses (normal and shear). In particular for shear stresses, I use a 3-boom approximation for the skin and spar. The code works for 1 spar at the moment.

Structural Fabrication: Wing manufacturing ongoing...

Vector Field Path Following for MAVs
Advisors: Prof. S. Ghosh

For Miniature Air Vehicles (MAVs) windspeeds are a major concern with respect to path following, as they can be upto 20-60% of the desired airspeed. Thus, wind can lead to significant deviations from a desired path.

In this paper we focus on path tracking (not trajectory tracking) and use a vector field guidance law with a sliding mode control approach to bring an MAV to the desire path and orient it along it.

In particular, we construct 2 types of vector fields; one for straight line path following and another for orbit (or circular arc) following. As an example, the figure on the left shows a vector field that guides an MAV to track a straight line path.

The algorithm is based on this paper.

Hurst Exponent - MFDFA
Colleague: Dr. I. Pavithran

Hurst exponent is a measure of long-term memory of time series. I was using the Hurst Exponent as an Early Warning Signal (EWS) for my research work. It took me a really long time to get everything working, so I decided to release my code on MATLAB file exchange :)

The code uses the Multifractal Detrended Fluctuation Analysis (MFDFA) algorithm which is based on this paper.

Algorithm: What we do is break the time series into parts of equal length (the length of each part is a scale), then fit a trend (linear or other) to each part and calculate the rms of the error of each part. Adding all the erros gives us the total error at that chosen scale. If we plot the total error as a function of the scale we get a plot, the slope of which is the Hurst exponent.

The toolbox now has over 80 downloads.

Anime Face Recognition Neural Network
MOOC: Stanford Machine Learning

I was studying machine learning and a course project was to build a neural network which would train on a dataset of handwritten numbers.

I wanted to make this more complex, and since I had a lot of free time due to college being off due to COVID, I decided to make a face identifier. But, I didn't have enough images of me or any of my friends to train the model. So, instead I used anime faces.

But even that was an issue since it is difficult to find face images of the same character, of a given size. So I ended up making a shabby 50x50 image size dataset of my own ;). I took 2 popular characters and edited their images (bitmap) to produce the dataset.

The final model had 1 hidden layer and when trained identified new images correctly with a 97% accuracy.

Image Filters
MOOC: Harvard CS50

I was learning C and I was learning about header files and how to make projects. Basically, I wanted to go beyond single file projects. I had also learnt about function pointers which was a pretty cool concept (until then I thought this was a thing only in Python).

Sitting at home due to COVID, I had written codes for different image filters inspired by a project I did in the online CS50 course. So I combined my filter codes with my desire to make a full blown project in C and ~Ta-Dah

This code works on bitmap images, which store each individual pixel color values. Each filter reads individual pixel which is a set of R G B values. Then these values are manipulated to produce the desired result.

For eg: The images on the left are converted using an invert filter. This is the invert filter, in which the R G B values of each pixel are taken and subtracted from 255 to give the new R G B values which then produce the new image.

Some of the image filters which I coded are:

Boundary - Detects boundaries in the images by finding pixel color derivatives
Reflect - Does a mirror reflection on the image
Grayscale - Produces a black and white image
Blur - Produces a blurred version of the original image
Invert - Takes the R G B value of each pixel and subtracts from 255 -> My Fav
Whiten - Increases the image brightness
